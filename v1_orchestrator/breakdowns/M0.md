### M0-T3 · ca-lib core types (serde-serializable shared messages)

**Deliverable.** Public types in `ca-lib`: `Architector`, `Task`, `Commit`, `ReviewResult`, `CritiqueResult`, `RpcRequest`, `RpcResponse`. All `Serialize + Deserialize + Debug + Clone + PartialEq`. Doc comments on every public item. Round-trip serde tests for each type.

**Expectation.** `cargo test -p ca-lib` passes with ≥6 round-trip tests. `cargo doc --no-deps -p ca-lib` builds with no warnings. Other crates can `use ca_lib::{Task, Architector, ...}`.

**Scope.**
- New: `crates/ca-lib/src/{architector.rs, task.rs, commit.rs, review.rs, rpc.rs}` (or one `types.rs` if you prefer; pick one and document)
- Modified: `crates/ca-lib/src/lib.rs` to re-export the types
- Modified: `crates/ca-lib/Cargo.toml` to depend on serde + serde_json
- Out of scope: RPC dispatch logic (M1), DB schema (M1)

**Motivation.** Types are contracts. Shared between daemon, task-processor, CLI, TUI. Locking them in early prevents protocol drift across crates.

**Validation.**
- Each public type has a `#[derive(Serialize, Deserialize, Debug, Clone, PartialEq)]`
- Each public type has a round-trip test
- `RpcRequest` is an enum with `#[serde(tag = "type")]` so JSON has a discriminator
- Unknown enum variants produce an error (no silent drop)

**Test scenarios.**
- _unit_ — **architector_serde_roundtrip**: serialize an `Architector`, parse, assert equal
- _unit_ — **task_serde_roundtrip**: same for `Task`
- _unit_ — **commit_serde_roundtrip**: same for `Commit`
- _unit_ — **review_result_serde_roundtrip**: same for `ReviewResult`
- _unit_ — **critique_result_serde_roundtrip**: same for `CritiqueResult`
- _unit_ — **rpc_unknown_variant_errors**: deserializing `{"type":"NonExistentRequest"}` returns an `Err`, not `Ok`

**Blockers:** M0-T2 merged
**Estimated LOC:** ~150

### M0-T4 · Lint/format configs + GitHub Actions CI

**Deliverable.** `rustfmt.toml`, `clippy.toml`, `.github/workflows/ci.yml`. Workflow gates on push + PR: `cargo fmt --check`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace --all-targets`. Project `CLAUDE.md` updated to reference these gates as the completion rules.

**Expectation.** A PR with any of (clippy warning / fmt violation / failing test) fails CI. Required for merge.

**Scope.**
- New: `rustfmt.toml`, `clippy.toml`
- New: `.github/workflows/ci.yml` (Rust toolchain + cache + 3 jobs or 1 job with 3 steps)
- Modified: `CLAUDE.md` adds the gates as the completion rules
- Out of scope: deploy infra, additional CI jobs (security scan, etc.)

**Motivation.** Every later PR ships through these gates. Without them no signal on regressions.

**Validation.**
- Push to a known-good commit → CI green
- Push a side-branch commit with `let _x = unused_value;` → CI fails at clippy
- Push a side-branch commit with malformed indentation → CI fails at fmt
- All 3 gates run as required checks; merge button is disabled if any fail

**Test scenarios.**
- _integration_ — **ci_passes_on_clean_commit**: HEAD on main passes all three CI gates
- _integration_ — **ci_blocks_clippy_warning**: side branch with a clippy warning fails CI; removing the warning turns CI green
- _integration_ — **ci_blocks_fmt_violation**: side branch with bad fmt fails CI

**Blockers:** M0-T2 merged
**Estimated LOC:** ~60
