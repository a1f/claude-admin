Rewrite claude_admin to a web admin orchestrator

Goal

Replace the current Rust TUI/CLI/daemon stack with a self-hosted web admin tool that orchestrates Claude (and Codex) work across all my projects — from planning, through breakdown, dispatch, review, critique, validation, all the way to merge.

Motivation

Today: I plan in Notion, break down in markdown, dispatch by hand in tmux, review with my eyes, merge by typing. Each step lives in a different tool. There's no audit trail of why the architect made a decision, no central place to see what every running session is doing, and no way to enforce that a PR has validation evidence before I'm asked to approve it.

Tomorrow: one URL. I open it, I see everything, I read the architect's "ready to merge" explanation, I click yes, it merges. Plans live in the same tool that dispatches their PRs.

Validation strategy

How we know we got it right:

  1. Self-rewrite test. By M9, claude_admin v2 ships its own milestones through itself. The remaining milestones (M10+) are dispatched, reviewed, critiqued, and merged with no terminal-side actions on my part.
  2. Side-by-side dogfooding. M1 sessions dashboard runs alongside the existing TUI for two weeks. If the TUI never gets opened, M1 is good.
  3. Reviewer/critic precision. Track for the first 10 PRs how often the architect's decision matched my override (after I look). Target ≥80% by end of M6.
  4. Validation gate. Every UI PR after M7 produces ≥1 screenshot artifact before approval is allowed. Auditable in artifact gallery.
  5. Final critique catches gaps. At least one milestone where the final critique surfaces a gap I'd have missed and converts it to a follow-up PR.

How we prevent regression

  • Every PR ships with new tests — coder is prompted explicitly. Critic checks acceptance evidence.
  • Replays: every architect decision is reproducible from inputs in the DB. Bisect bad decisions.
  • Daemon disconnect tests in CI from M0; chaos hour weekly.
  • Cost ceilings on every agent role; alert if a run exceeds 2× median.
