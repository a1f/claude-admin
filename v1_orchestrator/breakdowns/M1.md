## Tasks

### M1-T1 · ca-daemon tokio runtime + UDS bind + graceful shutdown

**Deliverable.** `ca-daemon` binary that on `cargo run -p ca-daemon` enters a tokio runtime, creates `~/.work/ca.sock` (mode 0600), accepts connections (no RPC yet — just connects, immediately closes with a placeholder banner line), logs every accept via `tracing`. Drains on SIGTERM / SIGINT and removes the socket file before exit. Refuses to start if the socket already exists (no silent overwrite).

**Expectation.** After this PR: `cargo run -p ca-daemon` runs in foreground; another shell can `nc -U ~/.work/ca.sock` and connect (gets a banner line). Ctrl-C in the daemon's shell exits cleanly within 1s; the socket file is gone afterward.

**Scope.**
- Modified: `crates/ca-daemon/{Cargo.toml, src/main.rs}`
- New: `crates/ca-daemon/src/socket.rs` (bind + accept loop + signal handling)
- Out of scope: any RPC framing (T2), DB (T3), CLI client (T4)

**Motivation.** Foundation for everything M1+. Get the socket lifecycle right before piling protocol on top.

**Validation.**
- Daemon starts; `~/.work/ca.sock` exists with permissions `0600`
- `nc -U ~/.work/ca.sock` connects and reads a banner ("ca-daemon X.Y.Z proto=1\n" or similar)
- SIGTERM (and SIGINT) exit within 1s; socket file removed
- Starting a second daemon while one is alive errors clearly with "socket already exists at … (is another daemon running?)"

**Test scenarios.**
- _integration_ — **daemon_creates_socket_on_start**: launch daemon as subprocess, assert socket exists with mode 0600
- _integration_ — **daemon_removes_socket_on_sigterm**: launch, signal SIGTERM, assert socket file gone after exit
- _integration_ — **daemon_refuses_to_start_if_socket_exists**: pre-create the socket, daemon errors with non-zero exit + clear message

**Blockers:** none
**Estimated LOC:** ~120

### M1-T2 · JSON-RPC framing + protocol version handshake

**Deliverable.** Newline-delimited JSON over the UDS. Daemon dispatches `RpcRequest` → `RpcResponse` using ca-lib types. Two requests implemented: `Version` (returns daemon version + protocol = 1) and `Ping` (returns `Pong { uptime_s }`). Unknown variants return `RpcResponse::Error`. Malformed JSON returns `Error` and the daemon stays up for the next connection. Per-connection handler is `tokio::spawn`'d so multiple clients are concurrent.

**Expectation.** After this PR: `echo '{"type":"version"}' | nc -U ~/.work/ca.sock` returns `{"type":"version","daemon":"0.1.0","protocol":1}`. `echo '{"type":"ping"}' | nc -U ~/.work/ca.sock` returns `{"type":"pong","uptime_s":N}`. Three concurrent `nc` clients each get their own response.

**Scope.**
- New: `crates/ca-daemon/src/rpc.rs` (per-connection handler loop)
- Modified: `crates/ca-daemon/src/{main.rs, socket.rs}` (accept loop spawns `rpc::handle_connection`)
- Modified: `crates/ca-daemon/Cargo.toml` (add serde_json + ca-lib)
- Out of scope: DB (T3), CLI client (T4), all other RPCs (only Ping + Version in this PR)

**Motivation.** Protocol contract before any business logic. Versioning the protocol now means M2+ can evolve it safely without breaking the CLI.

**Validation.**
- Version handshake returns daemon version + protocol = 1
- Ping returns `uptime_s` as a u64 ≥ 0
- Unknown variant returns Error variant; connection closes cleanly after
- Malformed JSON returns Error; daemon stays up
- 3 concurrent clients get correct independent responses
- Connection-handler errors don't kill the accept loop

**Test scenarios.**
- _integration_ — **version_handshake_returns_daemon_and_protocol**: connect, send version request, parse response, assert daemon == ca-lib version + protocol == 1
- _integration_ — **ping_returns_pong_with_uptime**: send ping, parse pong, assert uptime_s >= 0
- _integration_ — **unknown_request_returns_error**: send `{"type":"non_existent_request"}`, get Error response with clear message
- _integration_ — **malformed_json_returns_error_not_crash**: send `not json`, get Error, daemon still alive for a follow-up request
- _integration_ — **concurrent_clients**: 3 simultaneous connections, each gets its own correct response

**Blockers:** M1-T1 merged
**Estimated LOC:** ~180

### M1-T3 · SQLite schema + sqlx-migrate

**Deliverable.** SQLite at `~/.work/ca.db`. Four tables: `architectors`, `tasks`, `dispatches`, `reviews` (minimal columns matching ca-lib types). sqlx migrations under `crates/ca-daemon/migrations/0001_init.sql`. Daemon opens the DB on startup, runs migrations (idempotent), logs `DB ready · N architectors · N tasks` via tracing.

**Expectation.** After this PR: `cargo run -p ca-daemon` creates `~/.work/ca.db` if missing and applies migrations. Subsequent start is a no-op. Tables exist (`sqlite3 ~/.work/ca.db ".schema"`). Startup log includes the row counts line.

**Scope.**
- New: `crates/ca-daemon/migrations/0001_init.sql`
- New: `crates/ca-daemon/src/db.rs` (Pool setup + migrate + small select/insert helpers for Architector + Task)
- Modified: `crates/ca-daemon/src/main.rs` (open DB before binding socket; pass `Pool` into the rpc handler in T2's structure)
- Modified: `crates/ca-daemon/Cargo.toml` (add sqlx with sqlite + macros + migrate features — already declared at workspace level)
- Out of scope: RPCs that use the DB (M2+); the DB is opened and ready, but Ping/Version don't touch it yet

**Motivation.** Persistent state from M1 means daemon survives restart with full context. Schema laid out now so M2+ inserts are simple.

**Validation.**
- Fresh start creates DB with all 4 tables
- Re-start is a no-op (no errors, no double-creates)
- `sqlite3 ~/.work/ca.db ".schema"` shows architectors / tasks / dispatches / reviews
- Startup log shows the counts line
- DB has `pragma foreign_keys = ON` enabled

**Test scenarios.**

### M1-T4 · ca CLI subcommands + retry+backoff client

**Deliverable.** `ca` CLI has 3 subcommands: `ca daemon` (runs the daemon — currently delegates to `ca-daemon` main), `ca ping` (one-shot probe), `ca version`. The CLI client connects to `~/.work/ca.sock` with retry + exponential backoff (3 attempts: 0.5s / 1s / 2s). On success: prints `pong (uptime Ns)` and exits 0. On failure: prints `daemon not reachable at ~/.work/ca.sock (after N retries)` to stderr and exits 1.

**Expectation.** After this PR: `ca daemon &` then `ca ping` in another shell prints "pong" and exits 0. `ca version` prints `ca-daemon 0.1.0 · protocol 1`. With no daemon running, `ca ping` retries 3 times (~3.5s total) then errors clearly.

**Scope.**
- Modified: `crates/ca/src/main.rs` (clap-based dispatcher; was placeholder)
- New: `crates/ca/src/client.rs` (UDS client with retry+backoff)
- Modified: `crates/ca/Cargo.toml` (add tokio, clap, ca-lib, serde_json, anyhow, tracing)
- Out of scope: all other subcommands (architect register, task list, etc. — M2+)

**Motivation.** Closes the M1 loop end-to-end. After this PR, a real shell session verifies the daemon works without us hand-crafting `nc` invocations.

**Validation.**
- `ca daemon` starts daemon; from another shell `ca ping` prints "pong" + exits 0
- `ca version` prints daemon + protocol versions in one line
- No daemon running: `ca ping` retries 3 times, errors clearly within ~3.5s
- `cargo run -p ca -- ping` (cargo run path) also works

**Test scenarios.**
- _integration_ — **cli_ping_returns_pong_when_daemon_alive**: launch daemon subprocess, run `ca ping`, assert exit 0 + stdout contains "pong"
- _integration_ — **cli_ping_errors_when_daemon_down**: ensure no daemon, run `ca ping`, assert exit 1 + stderr contains "not reachable"
- _integration_ — **cli_version_matches_daemon_version**: launch daemon, run `ca version`, assert stdout matches `ca-daemon \d+\.\d+\.\d+ · protocol \d+`
- _integration_ — **cli_retries_3_times_with_backoff**: no daemon, time `ca ping`, assert duration in [3.0, 4.5]s (covers 0.5 + 1 + 2 backoff)

**Blockers:** M1-T1 merged; M1-T2 merged; M1-T3 merged
**Estimated LOC:** ~200
