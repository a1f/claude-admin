# PR4 — Plan history (versioning + diff)

## Goal
Every plan save creates a new immutable version row. Allow user to view diffs
between any two versions and revert to a prior version.

## Acceptance criteria
1. AC-1 — saving a plan creates a `plan_versions` row with full body snapshot.
2. AC-2 — version list shows author, timestamp, message (auto if blank).
3. AC-3 — selecting two versions shows a side-by-side diff (lines + sections).
4. AC-4 — "revert to version X" creates a new version that matches X (no destructive history).
5. AC-5 — the current 'live' plan is always the latest version.

## Reuse / refactor
- Reuse `MarkdownEditor` from PR2 for the diff renderer.
- Extract `useMarkdownDiff` hook — likely shared with future PR for issue diffs.

## Test plan
- Unit: `plan_version_service.spec` — create, list, diff, revert.
- Integration: e2e Playwright flow (save 3 times, diff v1↔v3, revert).
- Migration: smoke-test on existing plans (data backfill: each becomes v1).
